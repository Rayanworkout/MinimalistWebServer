A Minimalist Web Server
It works!

This is the default welcome page used to test the correct operation of this web server.

This project is not production ready. I made it for educational purposes.

If you can read this page, it means that the MinimalistWebServer installed at this site is working properly. You should replace this file (located in MinimalistWebServer/BaseServer/default.html) before continuing to operate your HTTP server.

How to use ?

This server is able to serve your static files. Simply create a project folder along MinimalistWebServer.py, launch the server and access the right URL. All libraries used are built-in inside Python 3; no external dependencies are needed.

You can organize your project folder as you want, but the recommended structure is as follows: 

MinimalistWebServer/your_project/
                        |-- index.html # or the name you want
                        |
                        |-- static/
                        |         -- css/
                        |               style.ss
                        |         -- img/
                        |               1.png
                        |               2.png
                        |               3.png
                        |         -- js/
                        |              index.js
  • When your project folder is ready, you can run the server with: 
    python3 MinimalistWebServer.py
    or under Windows: 
    python MinimalistWebServer.py
    The server listens on localhost:8080 by default, but you can mention another host / port when creating the server instance or at runtime. The port number must be of type int. and host of type str.

    At runtime: 
    python3 MinimalistServer.py --port 8000 --host localhost
    or at server creation, at the end of MinimalistServer.py
    server = MinimalistWebServer(port=8000, host="localhost")
  • Now the server is running, just access the URL displayed in your terminal (host address + port number).
    You should see this welcome page, unless you replaced default.html with your own.
  • You need to mention your html file in the path, otherwise the server won't find it. For example:
    http://host:port/your_project/index.html
  • That's it, you should be able to see your html file and all the linked static files will be served as well.
Note for Linux users

By default, Linux does not allow access through the web browser to any file apart of those located in /var/www, public_html directories (when enabled) and /usr/share (for web applications). If your site is using a web document root located elsewhere you may need to modify your files permissions accordingly.
